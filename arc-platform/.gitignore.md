# .gitignore Reference

## Protected Files

This project's `.gitignore` protects the following sensitive files from being committed:

### Environment Variables
- `.env` - Main environment file
- `.env.local` - Local overrides
- `.env*.local` - All local environment files
- `.env.development` - Development environment
- `.env.production` - Production environment
- `.env.staging` - Staging environment
- `.env.test` - Test environment

### API Keys and Secrets
- `*.key` - Private key files
- `*.secret` - Secret files
- `secrets.json` - JSON secrets
- `config/secrets.*` - Secret configurations

### Database Files
- `*.db` - SQLite database files
- `*.sqlite` - SQLite database files
- `*.sqlite3` - SQLite database files
- Database journal and WAL files

### Certificates
- `*.pem` - PEM certificates
- `*.crt` - Certificate files
- `*.cer` - Certificate files
- `*.p12` - PKCS#12 files
- `*.pfx` - PFX files

### Build and Cache
- `.next/` - Next.js build output
- `node_modules/` - Dependencies
- `*.tsbuildinfo` - TypeScript build info
- `next-env.d.ts` - Next.js types

### IDE and OS Files
- `.vscode/settings.json` - VS Code settings
- `.idea/` - IntelliJ IDEA
- `.DS_Store` - macOS
- `Thumbs.db` - Windows

### Logs and Temporary Files
- `*.log` - Log files
- `tmp/` - Temporary directory
- `temp/` - Temporary directory
- `*.tmp` - Temporary files

## Safe to Commit

These files are safe to commit:
- `.env.example` - Template with placeholder values
- `SECURITY.md` - Security documentation
- `SETUP.md` - Setup instructions
- `.gitignore` - This file

## Quick Commands

### Check if a file is ignored
```bash
git check-ignore -v filename
```

### List all ignored files
```bash
git status --ignored
```

### Force add an ignored file (NOT RECOMMENDED)
```bash
git add -f filename
```

## What to Do If You Accidentally Commit Secrets

1. **Immediately revoke the exposed credentials**
2. **Remove from git history:**
   ```bash
   git filter-branch --force --index-filter \
     "git rm --cached --ignore-unmatch path/to/file" \
     --prune-empty --tag-name-filter cat -- --all
   ```
3. **Force push (if already pushed):**
   ```bash
   git push origin --force --all
   ```
4. **Generate new credentials**
5. **Update environment variables**

## Prevention Tips

1. Always use `.env.local` for local development
2. Never use `git add .` without checking `git status` first
3. Use `git add -p` for interactive staging
4. Set up pre-commit hooks to scan for secrets
5. Use a secrets scanner like `git-secrets` or `truffleHog`
